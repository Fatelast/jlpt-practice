# AGENTS.md

## 核心理念与原则

- 简洁至上：遵循 KISS 原则，优先简洁、可维护，避免过度工程化和不必要的防御性设计。
- 深度分析：基于第一性原理分析问题，并善用工具提升效率。
- 事实为本：以事实为准，发现谬误时应坦率指出。

## 开发工作流

- 渐进式开发：通过多轮对话迭代明确并实现需求。
- 编码前必须先阅读相关代码、配置和已有实现。
- 涉及接口字段、数据结构、页面交互、兼容性、验收标准的关键疑点，必须先向用户确认。
- 不影响主流程的小疑点，基于项目现有约定做最小合理假设，并在最终说明中列出。
- 严格遵循“构思方案 -> 提请审核 -> 分解为具体任务”的作业顺序。
- 构思方案时优先检索和复用项目现有公共组件、工具函数、自定义 hooks 或全局状态，避免重复造轮子。
- 完成阶段性重大需求或重构后，主动沉淀项目知识快照。

## 输出规范

- 所有回复、思考过程及任务清单使用中文。
- 固定要求：Implementation Plan, Task List and Decision Notes in Chinese。

## 前端代码规范

- 遵循 Airbnb JavaScript Style Guide。
- 遵循 ESLint 规范，并结合 TypeScript/React/Vue 最佳实践。
- 展示中文文本需要使用项目国际化方法包裹，例如 `t('原中文文本')` 或 `this.$t('原中文文本')`。
- 禁止自行生成无意义 key。

## 注释规范

- JSDoc 只用于公共工具函数、复杂业务函数、跨模块复用函数、非显而易见逻辑。
- 简单事件处理、模板绑定、直观 getter/setter 不机械加注释。
- 复杂或 Hack 逻辑必须说明为什么这么做，而不是只描述做了什么。

## 文件操作

- 执行任何删除项目文件的 terminal 指令前必须向用户确认。
