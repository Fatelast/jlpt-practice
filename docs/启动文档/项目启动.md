# PROJECT_STARTUP.md

# 日语考试刷题小程序项目启动文档

## 1. 项目名称

```txt
jlpt-practice
```

## 2. 项目定位

本项目是一个 **日语考试刷题小程序**，专注 JLPT 备考刷题。

项目只做刷题相关能力，不做泛学习功能。

### 2.1 核心目标

- 提供 JLPT N5 / N4 起步的刷题功能
- 支持不同考试题型的专项练习
- 支持模拟测试
- 支持错题本
- 支持收藏题
- 支持答题记录
- 支持题目解析
- 支持题目反馈
- 支持管理后台维护题库

### 2.2 当前阶段暂不做

- 背单词
- 单词本
- 单词记忆曲线
- 课程学习
- 社区交流
- 支付会员
- AI 自动出题并直接上线
- 听力题正式功能

### 2.3 听力模块处理原则

听力题由于涉及音频资源生产、版权、上传、播放和存储成本，MVP 阶段暂不开放。

但数据库和类型设计中可以预留以下字段：

- `audioUrl`
- `audioText`
- `listening`

前端暂不展示听力入口。

后端暂不实现听力资源上传能力。

```ts
export type QuestionType =
  | 'single_choice'      // 普通单选
  | 'kanji_reading'      // 汉字读音

  | 'paraphrase'         // 近义表达

  | 'sentence_order'     // 句子排序
  | 'text_grammar'       // 文章语法
  | 'reading_choice'     // 阅读选择
  | 'listening_choice'   // 听力选择，MVP 预留
```

```ts
    | 'text_grammar'

  passage?: string;
```

# 12. 本地开发启动方式

## 12.1 启动数据库

建议使用 Docker 启动 MySQL。

根目录创建 `docker-compose.yml`：

```yaml
version: "3.8"

services:
  mysql:
    image: mysql:8.0
    container_name: jlpt-practice-mysql
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: password
      MYSQL_DATABASE: jlpt_practice
      MYSQL_USER: jlpt_user
      MYSQL_PASSWORD: jlpt_password
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    command:
      --default-authentication-plugin=mysql_native_password
      --character-set-server=utf8mb4
      --collation-server=utf8mb4_unicode_ci

volumes:
  mysql_data:
```

启动：

```bash
docker compose up -d
```

# 14. Codex 开发约束

## 14.1 总体原则

```txt
1. 先完成主流程，不要过早做复杂封装。
2. 保持前后端分离。
3. 小程序端不能直接访问数据库。
4. 数据库迁移脚本放在 server/prisma 中。
5. 所有接口统一返回 ApiResponse 格式。
6. 所有业务枚举统一定义。
7. 所有题目必须记录来源。
8. MVP 阶段不做听力正式入口。
9. MVP 阶段不做背单词相关功能。
10. 不要引入支付、会员、社区等非核心功能。
```

------

## 14.2 前端开发约束

```txt
1. 页面只处理展示和交互。
2. 接口请求统一放到 services。
3. 状态管理统一放到 stores。
4. 类型定义统一放到 types。
5. 通用组件放到 components。
6. 不要在页面里堆大量业务逻辑。
7. 做题页需要支持不同题型渲染。
8. 听力题类型可以保留，但入口隐藏。
```

------

## 14.3 后端开发约束

```txt
1. 使用模块化结构。
2. Controller 只处理入参和返回。
3. Service 处理业务逻辑。
4. Prisma 只在 Service 层调用。
5. 所有写操作需要校验权限。
6. 管理后台接口必须校验管理员身份。
7. 小程序端只允许获取 published 状态题目。
8. 删除题目优先使用软删除。
9. 答题记录需要保存 correctAnswer 快照。
10. 用户答错后需要写入或更新 wrong_questions。
```

------

## 14.4 数据库设计约束

```txt
1. 使用 MySQL。
2. 使用 utf8mb4 字符集。
3. 题目和选项拆表。
4. 标签和题目多对多。
5. 用户和错题多对多。
6. 用户和收藏多对多。
7. 练习记录和答题明细一对多。
8. 历史答题记录不要因为题目修改而丢失。
9. 题目来源字段不能为空。
10. 题目状态字段不能为空。
```
